CobWebKit

The world's worst rendering engine

CobWebKit is a custom written rending engine for HTML and CSS. It includes parsers for both languages, and outputs its graphical display to a JPanel.

Capabilities

CobWebKit is able to perform a number of functions, including:

  • Parse simple HTML pages into a tree.
  • Store HTML attributes such as id as part of the node.
  • Place the title tag content into the window title.
  • Read CSS from any style tags.
  • Parse the CSS, along with the default stylesheet into a list of selectors and rules.
  • Identify multiple separate selectors as well as ids, classes and tags from the selector
  • Apply the CSS rules to the correct elements in the tree and their children.
  • Render each element with the correct CSS styles, wrapping paragraphs as necessary.

Supported Styles

CobWebKit supports the following CSS properties:

  • color
  • font-size
  • font-family
  • font-weight
  • font-style
  • margin-top
  • margin-left
  • margin-right
  • margin-bottom
  • text-decoration

They do not display in quite the same way as they would in an actual browser, but do perform the same core functionality as the browser equivalents.

Limitations

In order to keep the project feasable for a single week, there are a few limitations

Parsing Limitations

All HTML elements can either contain text or other nodes, but not both. There is also no support for self closing tags, such as img tags.

The CSS parser is closer to the CSS parser a browser would use, but does not offer support for any selectors that use newer CSS features.

Neither parser does any kind of validation - they both assume that the input string is valid.

Other limitations

There is no support for CSS selector specificity, so a selector that applies to one element immediately overrides any rules that came before it. This applies to the children of elements too, so writing body {color: white;} would make the whole screen blank.
